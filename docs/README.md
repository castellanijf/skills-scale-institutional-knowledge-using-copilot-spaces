# OctoAcme Project Management Docs

This folder contains OctoAcme's project management process documentation, templates, and guides. Use this README as the single entry point to find process documents, quick summaries, and links to the full guidance stored in this repo's docs/ directory.

## Quick project management summary

- Initiation: Create a Project One-pager to confirm the problem, stakeholders, success metrics, and a go/no-go decision.
- Planning: Run a kickoff with stakeholders and the delivery team, prioritize the backlog, estimate scope, and create a release/milestone plan.
- Execution & Tracking: Follow a regular team rhythm (daily standups, weekly delivery syncs), manage work on the project board (Backlog → Ready → In Progress → In Review → QA → Done), and follow PR and CI conventions for safe, reviewable changes.
- Release & Deployment: Satisfy pre-release requirements (passing CI and security scans, release notes, rollback plan), deploy through staging with smoke tests, and use the rollback/incident playbook if needed.
- Retrospective & Continuous Improvement: Run retros after sprints/releases, convert learnings into action items, and track them in the backlog.
- Risk Management & Communication: Maintain a risk register, surface risks during weekly syncs, and escalate via defined paths (team → PM → Product Lead → Sponsor).
